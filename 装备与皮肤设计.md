# 仙侠塔防 · 装备与皮肤设计

> 配套《设计文档.md》v0.3 §9.2（装备）、§9.3（皮肤）、§9.5（Modifier 规范）、§10（Meta 进度）
> 本文件给出具体词条库（config 落地参考），所有 mod 必须遵守 §9.5 的运算规范与按 stat 分级封顶。

## 0. 设计原则

- **付费 ≠ 必胜**：所有付费装备/皮肤的属性加成总和，严格在 §9.5 封顶之内（伤害类 +150%、经济 +80%），且免费装备能凑到接近上限，保证竞技公平。
- **流派特化**：装备按剑/符/法/枪/阵分流派加成，鼓励玩家"配装"而非"全强"，增加 build 深度。
- **稀有度梯度**：凡(common) → 灵(uncommon) → 宝(rare) → 仙(epic) → 神(legend)，高稀有度数值更高且可能带"特殊词条"（非数值机制，如冷却缩减、双暴）。
- **槽位**：weapon（本命法宝）/ armor（护体法衣）/ accessory（随身灵物），每槽一件。
- **获取**：免费软货币（宗门贡献）抽卡为主，付费仙玉抽卡为辅（见 §10.3 寻仙）。

## 0.1 stat 命名约定（与 §9.5 对齐）

| stat | 含义 | 封顶类别 |
|---|---|---|
| `dmg` | 全塔伤害 | 伤害 +150% |
| `swordDmg`/`bowDmg`/`spearDmg`/`magicDmg` | 流派伤害 | 伤害 +150% |
| `rate` | 攻速 | 攻速 +60% |
| `range` | 射程 | 射程 +60% |
| `crit`/`critDmg` | 暴击率/暴击伤害 | 伤害（折算） |
| `bountyMul` | 赏金加成 | 经济 +80% |
| `slowPower` | 减速强度 | 攻速射程 +60% |
| `cooldown` | 技能冷却缩减 | 特殊，单独封顶 -40% |

## 4. 套装效果（可选深度，预留）

穿戴同系列 2 件/3 件触发套装加成（走 Modifier 管线的"套装来源"）。例：
- **青莲套装**（青莲剑 + 聚灵法衣）：2 件 → 剑修暴击伤害 +25%；
- **离火套装**（离火神扇 + 业火珠）：2 件 → 火法/雷法伤害 +20%。

> 套装为后期 build 深度，P5 可后置；结构上复用 Modifier 管线，零额外系统。

## 5. 皮肤设计

### 5.1 设计原则

- 皮肤**主要改外观**（icon、攻击特效、命中特效），属性为辅；
- **免费皮肤**：无属性或极低属性（+2%），通关/活动奖励；
- **付费皮肤**：带微量属性（+5%），但严格低于同级装备，避免"皮肤=装备强度"；
- 皮肤按塔绑定（`target`），一个塔可有多套皮肤。

### 5.3 皮肤扩展

- 每个塔预留至少 1 免费 + 1付费 + 1活动皮肤位；
- 活动/节日皮肤走限时 manifest（如"中秋·月华皮肤"）；
- 皮肤特效与战斗逻辑解耦：渲染层读 `visual.effect`，引擎不感知。

## 6. 与 Modifier 管线的对接（落地示例）

玩家装备 `{weapon: "xuanyuan", armor: "dragon_scale", accessory: "heaven_seal"}` + VIP2 + 飞剑皮肤 `fs_immortal` + meta天赋"剑修精通 +10%"，对**飞剑修士**的伤害结算（按设计文档 §9.5 跨 stat 组合规则，同性质伤害乘子加法合并）：

```
base = 塔自身配置 dmg

伤害类乘子（飞剑修士同时适用 swordDmg 与 dmg，加法合并为一个 effectiveDmgMul）：
  - 轩辕神剑   swordDmg +0.25
  - meta 剑修精通 swordDmg +0.10      → swordDmg Σ = 0.35
  - 龙鳞宝甲   dmg     +0.12
  - 天命玉玺   dmg     +0.10
  - fs_immortal dmg    +0.05          → dmg Σ = 0.27
  effectiveDmgMul = 0.35 + 0.27 = 0.62   （不是 1.35×1.27，而是加法合并）
  → 伤害类封顶 +150%，未触顶

final = base × (1 + 0.62) = base × 1.62
```

> 多来源叠加走 §9.5 公式：**同性质伤害乘子先加法合并再封顶**（T1 规则），加法叠加、按 stat 封顶，数值可控可预测。新增装备/皮肤/meta 只需产出 mod，不改结算。

## 7. 配置文件划分（config/）

```
config/
├── equipment.json     # §1-4 装备词条
├── skins.json         # §5 皮肤
├── sets.json          # §4 套装定义（可选）
└── gacha.json         # 寻仙抽卡池（配 equipment/skin 出率，见 §10.3）
```

每个条目带 `id / name / rarity / slot|target / mods / source`，`ConfigLoader` 启动时校验 mod 的 stat/op 是否合法、是否在封顶类别内（见设计文档 §3.1）。
